# 12 节启蒙课奖励与完成页文案实施计划

## 目标

把 12 节启蒙课的完成反馈收成正式产品口径：

- 第 3、6、9、12 节使用重完成页文案
- 其余 8 节使用统一的轻完成反馈
- 家长侧关键节点同步显示作品结果

## 实施顺序

### 任务 1：补齐完成页内容数据层

范围：

- 新增 4 个作品课节点的重完成页文案配置
- 新增 8 节普通课的轻完成反馈口径

落点：

- 现有项目完成页文案构建逻辑
- 现有 foundation lesson feedback 内容来源

验收：

- 4 个重完成页节点都能按 lessonId 读取 headline 和 summary
- 8 节普通课都能读取轻完成反馈

### 任务 2：接入运行时消费者

范围：

- 课程完成后读取对应完成文案
- 第 3、6、9、12 节显示作品导向的完成页
- 其余 8 节保留轻反馈

落点：

- project complete 页面相关文案读取逻辑
- lesson feedback 相关消费者

验收：

- 作品课完成时页面文案围绕“你做出了一个完整作品”
- 普通课完成时反馈语气统一

### 任务 3：家长侧同步描述

范围：

- 家长总览在 4 个关键作品节点显示明确作品结果

落点：

- parent overview 内容构建逻辑

验收：

- 第 3、6、9、12 节完成后，家长能看到对应作品描述

### 任务 4：测试与验证

内容测试：

- 4 个重完成页节点配置完整
- 8 节普通课轻反馈配置完整

运行时测试：

- project complete 页能读到重完成页文案
- 轻完成反馈消费者能读到正确文本
- 家长侧关键节点能读到作品结果

整体验证：

- `npm run test:run -- --maxWorkers=1`
- `npm run lint`
- `npm run build`
- 必要时补 `guest-story` 或家长侧 E2E

## 执行边界

本轮只做内容与文案结构，不扩：

- 新奖励机制
- 新卡片规则
- 新页面动画
- 新家长页结构
- 新付费逻辑
