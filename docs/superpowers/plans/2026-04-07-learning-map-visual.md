# 学习地图成长轨道视觉重构实施计划

## 目标

把学习地图页收成一条温暖、清楚、有方向感的成长路线：

- 顶部只强调当前单元和下一步
- 中央采用连续成长轨道
- `12` 个小站点和 `4` 个单元大里程碑层级清楚
- 终点自然接上高阶区域

本次不扩到首页、课程学习页、家长页和支付。

## 实施顺序

### 任务 1：重构学习地图页头

范围：

- 收紧页头信息层级
- 强调当前单元和下一步
- 弱化总进度数字和其他旁支信息

落点：

- `apps/web/src/app/learn/map/page.tsx`

验收：

- 一眼能看懂“现在在哪、下一步去哪”
- 页头不再像普通进度摘要区

### 任务 2：重构 MapView 主体轨道

范围：

- 把当前地图从普通列表或分散节点收成连续成长轨道
- 让轨道本身成为页面主角

落点：

- `apps/web/src/features/lessons/map-view.tsx`

验收：

- `12` 节启蒙课沿同一条轨道展开
- 当前节点和下一步节点明显可见

### 任务 3：定义节点与里程碑层级

范围：

- 小站点状态视觉区分
- 四个单元大里程碑表达
- 已完成、当前推荐、可进入、未到达状态收敛清楚

落点：

- `apps/web/src/features/lessons/map-view.tsx`
- 必要时相关样式和辅助组件

验收：

- 当前节点最突出
- 下一步节点次突出
- 已完成节点有积累感
- 单元节点像阶段成果，不像普通标题

### 任务 4：接入高阶区域的自然延伸

范围：

- 在启蒙轨道末端自然接出高阶区域
- 只表达“下一段成长”，不做强营销卡

落点：

- `apps/web/src/features/lessons/map-view.tsx`

验收：

- 高阶区域和启蒙轨道自然衔接
- 不破坏启蒙阶段主线

### 任务 5：测试与验证

组件测试：

- `MapView` 当前节点、单元里程碑、高阶区域展示
- 页头和地图结构消费者测试

整体验证：

- `npm run test:run -- --maxWorkers=1`
- `npm run lint`
- `npm run build`
- 必要时补地图或学习流相关 E2E

## 执行边界

本轮不做：

- 首页改版
- 课程页继续打磨
- 家长页改版
- 新支付逻辑
- 新奖励逻辑

本轮只解决：

“学习地图是否像一条真正的成长轨道，而不是课程入口列表。”
