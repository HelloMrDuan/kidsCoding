# 全站柔和梦幻系 3D 动画风视觉重构规格

## 背景

当前产品虽然已经完成了首页、学习地图、课程页、完成页、家长页、卡册页等主路径的一轮功能和文案收口，但整体视觉仍然停留在“网页卡片 + 教育信息排版”的层级。

这带来的问题很明确：

- 对孩子不够有吸引力，页面缺少统一的动画世界感
- 对家长不够高级，质感和品牌感不足
- 同一产品内，不同页面之间像不同阶段拼起来的结果，缺少统一视觉语言
- 学习主线虽然已经理顺，但视觉世界观还没有建立起来

用户已经明确要求：

- 视觉参考以国内优秀产品的页面信息清晰度为准
- 不再采用国外产品作为主要审美基准
- 风格改成“高品质动画片式”的 3D 卡通
- 不是玩具风，不是廉价游戏风，而是“柔和梦幻系”的中度 3D
- 不只改局部，而是全站可见页统一切换到同一套世界观

## 目标

本次重构的目标不是单纯“换皮”，而是把当前 Web 产品统一成一套完整的柔和梦幻系 3D 动画世界：

- 孩子一眼能感受到这是一个会陪他成长的动画学习世界
- 家长一眼能感受到产品有品质、可信、不是拼装页
- 首页、onboarding、学习地图、课程页、完成页、卡册页、家长页、购买/绑定页属于同一个世界
- 页面层级更清楚，每页只保留一个主操作，减少理解负担

## 非目标

本次重构不包含：

- 改动课程奖励发放逻辑
- 改动支付底层逻辑
- 改动 Supabase/AI/provider 等基础设施
- 引入 3D 引擎、WebGL 场景或重型动画库
- 重新设计信息架构以外的业务模型

本次只做：统一视觉系统、统一组件语言、统一页面表现。

## 用户与场景

### 孩子端

- 年龄：6-8 岁
- 核心诉求：看得懂、想点、知道下一步做什么
- 核心风险：如果信息太多、页面太平、世界观不统一，会觉得无聊或不知道如何操作

### 家长端

- 核心诉求：一眼看懂产品价值、学习主线、作品成果和下一步
- 核心风险：如果页面太游戏化、太散乱或太廉价，会降低信任感

## 视觉总方向

### 核心风格

全站采用：`柔和梦幻系的高品质 3D 动画风`

具体特征：

- 奶油色、云朵感、柔和景深
- 按钮、卡片、勋章、舞台都有圆润体积感
- 使用中度 3D：有厚度、有高光、有阴影，但仍然是高可用网页
- 角色、道具、页面模块像同一部动画片里的元素
- 页面不走游戏 HUD 风，也不走硬科技风

### 统一视觉母题

- 云层
- 柔和舞台灯光
- 动画片式高光
- 有厚度的收藏物件
- 圆润的场景边界
- 渐层景深背景

### 角色世界观

采用“主角群世界”：

- 小狐狸
- 蝴蝶
- 小鸟
- 其他动物朋友

不同单元轮流作为主角，但角色始终属于同一个世界。

这样可以同时满足：

- 世界统一
- 单元不重复
- 页面角色不再随机、错位、抽象化

## 信息与交互原则

### 一页一主操作

每个页面必须有且只有一个主操作按钮，其他动作全部降级：

- 首页：开始学第一节
- 年龄页：选择年龄并继续
- 准备页：开始进入学习地图
- 地图页：进入当前推荐课
- 课程页：完成当前步骤
- 完成页：回看作品
- 家长页：查看最近作品 / 继续孩子主线

### 结果优先

所有核心页都优先展示“结果”，而不是解释：

- 首页看动画作品舞台
- 课程页看舞台变化
- 完成页看作品回放
- 卡册页看收藏成果
- 家长页看最近作品和成长结果

### 说明降级

解释文案不能再像说明书一样挤在主要视觉区域里：

- 操作提示只说当前动作
- 补充说明移到次级区块
- 家长说明始终晚于孩子操作入口

## 视觉系统

### 颜色

使用统一色板：

- 主背景：奶油白、浅桃白、云雾蓝
- 主强调：暖橙、日光金
- 辅助强调：天蓝、草地绿、柔和紫
- 深色文字：深海军蓝，而不是纯黑

颜色分布规则：

- 大背景和面板：柔和浅色
- 主操作按钮：暖橙或暖金
- 当前高亮：天空蓝或暖光色
- 奖励/收藏：根据稀有度引入更强局部高光

### 阴影与立体

采用中度 3D 阴影体系：

- 卡片和按钮都具备柔和外阴影
- 高价值模块增加内阴影与高光边缘
- 舞台、勋章、收藏卡具备更强体积感

避免：

- 重金属、重玻璃拟物
- 过强投影导致廉价游戏 UI 感

### 组件语言

统一组件家族：

- 主按钮：圆润厚度按钮，具备动画片式亮面
- 信息卡：圆角厚边、内亮面、轻纹理
- 舞台：小剧场窗口语言
- 勋章：奖章陈列物件
- 卡片：收藏册页语言
- 地图节点：成长小站点

## 页面分层改造

### 1. 入口层

包括：

- 首页
- `/onboarding/age`
- `/onboarding/check`

要求：

- 首页像动画片海报 + 小剧场
- 年龄选择页像“开场选择”
- 准备页像“马上开场”的暖场页
- 视觉更梦幻，但结构更简单

### 2. 学习层

包括：

- 学习地图
- 课程页
- 微补课

要求：

- 地图像成长轨道
- 课程页像儿童创作工作台
- 舞台始终是视觉中心
- 任务卡、结果卡、微补课卡属于同一家族

### 3. 成就层

包括：

- 完成页
- 卡册页
- 勋章墙
- 星星总览

要求：

- 完成页像作品首映小舞台
- 卡册页像成长收藏馆
- 勋章墙像展馆陈列
- 星星、勋章、卡片归入同一套成长叙事

### 4. 家长层

包括：

- 家长总览
- 家长作品回看
- 购买/升级页
- 绑定账号页

要求：

- 保留同一套 3D 动画世界
- 装饰密度低于孩子页
- 结果和下一步更清楚
- 家长视图更稳、更可信

## 组件级改造要求

### 首页

- 第一屏：3D 作品舞台 + 单主按钮
- 第二屏：成长路径做成动画路线卡
- 第三屏：家长信任信息用更高级的 3D 信息牌
- 第四屏：CTA 像动画世界里的入口门牌

### 学习地图

- 连续成长轨道保持不变
- 节点改成更有体积感的小站点
- 单元里程碑像路标
- 高阶区域像轨道尽头发光的新大陆

### 课程页

- 中央舞台继续做大，但舞台改成更明显的 3D 小剧场
- 左侧任务卡改成动画片式任务面板
- Blockly 创作桌面继续保留，但边界和底板更像“工作台”
- 所有反馈卡统一成轻量立体卡

### 完成页

- 舞台是主角
- 奖励区更像陈列托盘
- 勋章和卡片出现方式更有收藏感
- 继续按钮不再抢主操作

### 卡册页

- 顶部成长总览做成展馆入口
- 勋章墙做成奖章陈列
- 收藏卡做成高质量收藏卡
- 页面背景要更像展馆空间，而不是普通白底

### 家长页

- 把“最近作品”做成主视觉区
- 把“下一步建议”做成更稳重的信息牌
- 作品回看页的舞台和说明要统一到同一套 3D 系统

## 实施顺序

为了尽快看到整体效果，执行顺序固定为：

1. 首页、学习地图、课程页、完成页
2. 卡册页、勋章墙、奖励相关页
3. 家长总览、作品回看、购买/绑定页
4. onboarding 与所有剩余入口页
5. 全站零散按钮、标签、空状态、表单外观统一

- 首页
- `/onboarding/age`

- 已完成课回学

### 自动化验证

继续保持：

- 相关组件测试
- 页面测试
- `lint`
- `build`
- `guest-story` 主路径 E2E

## 风险与控制

### 风险 1：只换皮，不解决理解负担

控制：

- 每页只保留一个主操作
- 说明降级
- 结果优先

### 风险 2：3D 过重导致页面更乱

控制：

- 采用中度 3D，不走重游戏 UI
- 结构优先于装饰

### 风险 3：页面之间风格不统一

控制：

- 统一颜色、阴影、按钮、卡片、舞台、勋章、地图节点语言
- 页面按同一世界观重做，而不是逐页自由发挥

## 结论

本次不是局部美化，而是一次明确的全站视觉重构：

- 目标风格：柔和梦幻系高品质 3D 动画风
- 审美参考：国内优秀儿童学习产品的信息清晰度
- 交互原则：每页一个主动作，结果优先
- 页面目标：把当前产品统一成一个可信、吸引孩子、家长也认可的动画学习世界
