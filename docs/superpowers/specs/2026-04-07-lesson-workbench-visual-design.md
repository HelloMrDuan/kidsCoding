# 课程页创作工作台视觉重构规格

## 1. 背景

当前项目已经完成：

- 首页学习主线重构
- `12` 节启蒙课四个单元的课程脚本
- `voiceover` 主语音骨架
- 单元级微补课
- 作品完成页和轻反馈卡的第一轮视觉升级

但课程学习页本身仍然存在几个核心问题：

1. 页面中心不够明确，舞台和结果感不够强
2. 任务区、舞台区、积木区的视觉层级仍然偏功能页
3. 页面更像“可用的学习界面”，还不像“高品质儿童创作工作台”
4. 完成页已经开始有产品感，但上课过程本身还没有完全接上这套视觉语言

这次重构的目标不是再加功能，而是把孩子真正上课时看到的主界面，拉到和首页、完成页一致的产品品质。

## 2. 设计目标

本次课程页视觉重构的目标如下：

1. 让课程页第一眼更像儿童创作工作台，而不是任务后台
2. 明确把作品舞台和创作结果放到视觉中心
3. 让孩子始终知道“我现在在做什么”
4. 让任务区更轻、更短、更不压迫
5. 让 Blockly 区域更像创作桌面的一部分，而不是技术工具面板
6. 让轻反馈、微补课、主语音等辅助信息和舞台体验属于同一家族

## 3. 参考原则

本次页面重构明确参考儿童编程和儿童创作产品的成熟规律，不闭门造车。参考重点不是照搬页面，而是提炼出已经被验证有效的交互规律：

1. 先看到结果，再理解步骤
2. 一次只给一个清晰小目标
3. 舞台和角色是页面主角，不是说明文字
4. 提示是支持性的，不是纠错式的
5. 反馈要嵌在学习流里，而不是不停打断

本次课程页主要借鉴的成熟规律包括：

- 大舞台优先的创作界面
- 低压迫的左侧任务卡
- 预览在上、创作在下的工作台结构
- 小而清楚的结果反馈卡
- 圆角、暖色、轻层次的儿童创作产品语言

## 4. 范围

本次重构只覆盖学习页工作台本身及其直接依赖组件：

- `apps/web/src/app/learn/lesson/[lessonId]/page.tsx`
- `apps/web/src/features/lessons/guided-lesson-shell.tsx`
- `apps/web/src/features/lessons/blockly/preview-stage.tsx`
- `apps/web/src/features/lessons/blockly/blockly-workspace.tsx`

必要时可同步调整直接服务于以上页面的样式、测试和壳层数据读取逻辑，但不扩到：

- 首页
- 学习地图
- 家长页整体布局
- 支付页面
- 后台

## 5. 页面结构

### 5.1 总体布局

课程页采用“轻左栏 + 大中栏”的工作台结构：

- 左侧：轻任务区
- 中间：大舞台和积木创作区
- 右侧：不保留强独立列，弱化信息密度，必要信息并入左侧和舞台下方

页面的视觉中心必须始终是中间创作区域，而不是任务说明。

### 5.2 左侧任务区

左侧不再承担复杂说明，而是承担“今天这一步要做什么”。

左侧任务区包含：

1. 当前任务卡
2. 当前步骤短说明
3. 主语音摘要或播放入口
4. 舞台下方的小结果卡
5. 微补课卡片

任务区设计原则：

- 一次只强调一个步骤
- 文案短
- 不堆积多层解释
- 让孩子看一眼就知道下一步

### 5.3 中央舞台区

中央区域是页面真正主角。

结构固定为：

1. 上方作品舞台
2. 下方 Blockly 创作区

舞台区要求：

- 更大
- 更亮
- 更有展示感
- 与完成页的舞台语言保持一致

这里不是一个普通预览框，而是“我的作品正在这里发生”。

### 5.4 Blockly 创作区

Blockly 区域不再强调“工具感”，而强调“创作桌面感”。

要求：

- 保持可读、清楚
- 弱化技术面板味道
- 与舞台区形成上下连贯结构
- 在视觉上像孩子正在操作一张创作桌，而不是在填表

## 6. 核心区块定义

### 6.1 当前步骤任务卡

当前步骤用任务卡表达，而不是长说明栏。

任务卡至少包含：

- 当前步骤标签
- 一句当前目标
- 简短说明

任务卡气质：

- 温暖
- 轻量
- 清楚
- 不压迫

### 6.2 结果反馈卡

轻反馈卡固定放在左侧任务区下方，不浮层、不弹窗。

内容组成：

- 一句当前结果
- 一个小预览或小结果提示

作用：

- 告诉孩子“这一步已经真的做成了”
- 不打断学习
- 让孩子感到作品正在长出来

### 6.3 微补课卡

微补课卡继续沿用已经做好的单元级微补课能力，但视觉上要和任务卡、反馈卡同一家族。

要求：

- 不像报错框
- 不像系统警告
- 更像老师走过来轻轻提醒

## 7. 视觉语言

### 7.1 气质

页面整体气质定义为：

- 儿童创作工作台
- 温暖
- 圆润
- 有舞台感
- 低压迫

避免：

- SaaS 仪表盘感
- 生硬的三栏后台感
- 冷白色教育官网感
- 过度游戏结算页感

### 7.2 色彩

课程页延续当前产品的温暖启蒙色系，但进一步收敛为：

- 温暖奶油底色
- 柔和橙色或蜜桃色强调
- 清新蓝绿色辅助舞台与互动
- 少量草地绿用于成长和完成状态

### 7.3 组件语言

统一组件语言：

- 大圆角
- 柔和阴影
- 清晰层级
- 少量轻动画

避免：

- 密集描边
- 复杂按钮堆叠
- 太多发光特效

## 8. 运行时行为要求

### 8.1 视觉优先级

课程页运行时的视觉优先级固定为：

1. 舞台结果
2. 当前任务
3. 创作操作
4. 反馈与微补课

### 8.2 步骤切换

步骤切换时要求：

- 当前任务卡更新
- 舞台持续可见
- 已有积木状态不被视觉重置感打断
- 页面感受是“继续创作”，不是“重新打开一个新页面”

### 8.3 反馈出现方式

轻反馈卡在操作成功后自然出现，不做高打断弹层。

微补课卡在孩子连续卡住时出现，优先帮助继续当前课，不把孩子跳出页面。

## 9. 非目标

本次不做：

- 课程机制重写
- 新增玩法系统
- 首页再次改版
- 学习地图再次改版
- 家长页布局重做
- 新支付逻辑
- 新奖励系统逻辑

本次只解决：

“孩子真正上课的主界面，看起来是否像一个正式、高品质、儿童友好的创作工作台。”

## 10. 实施要点

建议按下面顺序实现：

1. 先重构学习页整体布局
2. 再重构 `GuidedLessonShell`
3. 再升级 `PreviewStage`
4. 再把 `BlocklyWorkspace` 视觉收进同一家族
5. 最后补测试和 E2E

## 11. 验收标准

本次完成后，至少需要满足：

1. 课程页第一眼的视觉中心是作品舞台
2. 左侧任务区比现在更轻、更短、更清楚
3. 预览舞台在上、积木区在下的结构清晰成立
4. 轻反馈卡自然嵌在任务区下方，不打断学习流
5. 微补课卡和课程页视觉属于同一家族
6. 整体页面更像儿童创作工作台，而不是管理界面
7. `guest-story` 相关学习流 E2E 继续通过
8. `lint`、`test`、`build` 继续通过

## 12. 自检

已完成自检，确认如下：

- 范围只收在课程学习页及其核心依赖组件，没有扩到首页、地图和支付
- 设计方向与前面已完成的完成页视觉升级保持一致
- 已明确参考儿童编程和儿童创作产品的成熟规律，而不是主观拍板
- 页面目标始终围绕学习主线和作品结果，不让辅助信息抢主角
- 没有留 `TODO`、`TBD` 或模糊占位要求
